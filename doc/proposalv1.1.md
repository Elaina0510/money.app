## 1. 版本概述

### 1.1 版本目标

v1.1 的核心目标是 **修复 v1.0 的已知问题**，同时根据实际使用体验进行 **UI/UX 改进**，并引入少量必要的新功能。

| 类别 | 内容 |
|------|------|
| 🐛 Bug 修复 | 共 **6 项**（见第 2 节） |
| ✨ 新功能/改进 | 共 **5 项**（见第 3 节） |
| 🎨 界面变更 | 主题色、导航布局、时间显示等 |

### 1.2 与 v1.1 原计划的差异

原 v1.0 需求文档中规划的 v1.1 功能（账单导入、数据导出、数据备份恢复、统计图表增强）**推迟到后续版本**，本次聚焦于修复和打磨现有功能。

## 2. Bug 修复清单

### Bug 1：PC Web 端缺少导航栏（左侧边栏）

| 项目 | 说明 |
|------|------|
| **现象** | 在 PC 浏览器上访问时，没有任何导航栏（既无左侧边栏也无底部导航），无法切换页面。安卓端通过 WebView 访问时可正常显示左侧边栏。 |
| **原因推测** | 前端布局组件（`AppLayout.vue`）在响应式判断中，对 PC 宽屏的逻辑分支缺少左侧边栏的渲染，或 CSS/条件渲染未覆盖 PC 端场景。 |
| **修复要求** | PC 宽屏端显示左侧边栏导航，与手机端逻辑保持一致。 |
| **验收标准** | PC 浏览器（Chrome/Edge）上打开应用，左侧边栏可见，点击各菜单项可正确切换页面。 |

### Bug 2：首页最近账单显示为空

| 项目 | 说明 |
|------|------|
| **现象** | 首页（仪表盘）的"最近几笔账单"区域显示"还没有记账记录"，但进入账单列表页（RecordListPage）可正常显示所有记录。 |
| **原因推测** | 首页调用的接口与账单列表页调用的接口不同，可能首页调用了错误的 API 端点，或参数传递有误，或返回数据格式不匹配。 |
| **修复要求** | 首页正确获取并展示最近几笔账单记录。 |
| **验收标准** | 有记账数据时，首页显示最近 N 条记录；无数据时显示空状态提示。 |

### Bug 3：统计总金额始终显示为 0

| 项目 | 说明 |
|------|------|
| **现象** | 首页仪表盘的月度统计卡片和统计页面（StatisticsPage）的收支金额均显示为 0，即使数据库中有正常记账数据。 |
| **原因推测** | 统计数据相关的后端接口（`/api/statistics/`）或前端调用逻辑存在问题，可能 SQL 查询语句有误，或返回数据解析/展示环节出错。 |
| **修复要求** | 修复统计数据接口及前端展示逻辑，确保金额正确计算并显示。 |
| **验收标准** | 录入多条记账数据后，首页统计卡片和统计页面的总支出/总收入/结余等金额显示正确。 |

### Bug 4：深色模式下文字与背景颜色冲突

| 项目 | 说明 |
|------|------|
| **现象** | 深色模式下，部分页面的文字颜色与背景高度重合（如文字为深灰/白色，背景也为深灰/白色），导致文字难以辨认。 |
| **原因推测** | 前端 CSS 变量在深色模式下的值定义不当，或某些元素未正确应用深色模式的样式变量。 |
| **修复要求** | 调整深色模式下的颜色变量，确保所有界面元素的文字与背景有足够的对比度（符合 WCAG AA 标准，对比度 ≥ 4.5:1）。 |
| **验收标准** | 切换到深色模式后，所有页面的文字清晰可读，无颜色重合现象。 |

### Bug 5：PC Web 默认深色模式且无法切换

| 项目 | 说明 |
|------|------|
| **现象** | PC 浏览器上打开应用时默认进入深色模式，且界面中没有任何切换浅色/深色模式的控件。**手机端（安卓 WebView）正常**，有切换按钮且可正常切换。 |
| **原因推测** | 前端主题切换按钮在 PC 端的渲染条件缺失，导致 PC 端未显示切换控件；可能默认主题状态设置跟随系统，而 PC 系统为深色模式。 |
| **修复要求** | 见第 3 节 [新功能 2：深色/浅色模式切换](#新功能-2深色浅色模式切换)。 |

### Bug 6：账单详情页默认勾选，无法查看详情

| 项目 | 说明 |
|------|------|
| **现象** | 在账单列表页点击某条记录时，该记录被勾选（选中状态），而非进入详情页面查看完整信息。 |
| **原因推测** | 账单列表项点击事件绑定的行为是「选中/勾选」而非「查看详情」，缺少独立的详情查看功能。 |
| **修复要求** | 见第 3 节 [新功能 1：账单详情页](#新功能-1账单详情页)。 |

## 3. 新功能与改进

### 新功能 1：账单详情页

| 项目 | 说明 |
|------|------|
| **背景** | 当前点击账单列表项触发的行为是勾选，无法查看记录的完整信息。 |
| **需求** | 点击账单条目时，打开一个**详情页面/弹窗**，展示该条记录的完整信息，并支持编辑和删除操作。 |
| **详情页展示内容** | 金额、类型（收入/支出）、分类（以图标形式展示）、标签（作为主标题）、消费时间、附件（如有） |
| **交互** | - 详情页底部提供「编辑」和「删除」按钮<br>- 点击「编辑」跳转到编辑表单页（复用现有 RecordFormPage）<br>- 点击「删除」弹出确认对话框 |
| **验收标准** | 点击账单列表项 → 进入详情页 → 可查看完整信息 → 可跳转编辑或删除。 |

### 新功能 2：深色/浅色模式切换（PC 端补充）

| 项目 | 说明 |
|------|------|
| **背景** | 手机端已有切换按钮且功能正常，但 PC 端缺少切换按钮且默认深色。 |
| **需求** | 在 PC 端界面中增加**手动切换深色/浅色模式的开关按钮**，与手机端功能保持一致。 |
| **位置建议** | 左侧边栏底部或顶部导航栏右侧，使用太阳/月亮图标切换按钮（与手机端统一位置）。 |
| **行为** | - 切换后立即生效，PC 端和手机端逻辑一致<br>- 切换状态保存到 localStorage，刷新后保持用户选择<br>- 默认浅色模式（与手机端一致） |
| **验收标准** | PC 端出现切换按钮，点击可在深色/浅色之间切换；刷新后保持选择；手机端不受影响保持原有正常行为。 |

### 新功能 3：账单列表布局重构

| 项目 | 说明 |
|------|------|
| **背景** | 当前账单列表以"分类"（category）作为标题，用户希望改为以"标签"（tag）为主标题，分类以图标形式展示。 |
| **需求** | **账单列表每行记录重新布局如下：** |
| 布局示意 | `[分类图标] [标题]`<br>`金额  日期` |
| **说明** | - **主标题**：优先显示该记录关联的**标签（tag）**，如"奶茶"、"共享单车"；如果该记录**没有标签**，则主标题回退显示**分类（category）名称**<br>- **副标题位置**：显示金额和日期（仅日期，不显示时分）<br>- **前缀图标**：始终显示该记录所属**分类（category）** 的图标，如餐饮🍽️、交通🚗（使用 Material Icons）<br>- 一条记录限制一个标签，不存在多标签的情况 |
| **标签非必填** | 用户新增/编辑账单时，**标签为非必填字段**，**分类为必填字段**。 |
| **验收标准** | 有标签时：分类图标 + 标签标题 + 金额 + 日期；无标签时：分类图标 + 分类名称 + 金额 + 日期。 |

### 新功能 4：快速记账时标签自动匹配分类

| 项目 | 说明 |
|------|------|
| **背景** | 在新增记账页面，用户输入标签后希望能自动匹配对应的分类，减少手动选择分类的操作。 |
| **需求** | 用户在记账表单中输入或选择**标签**时，系统根据该标签自动匹配对应的**分类（category）**。 |
| **匹配规则** | - 根据标签与分类的关联关系，自动填充分类字段<br>- **仅匹配分类**，不自动填充金额和消费时间（金额和时间需用户手动填写）<br>- 如果该标签没有关联任何分类，则分类字段保持为空（用户仍需手动选择） |
| **实现方式** | 需要在标签（tag）数据模型中增加一个可选的关联分类字段（`category_id`），在用户创建标签时可选择关联到某个分类。后续新增记录时，选择标签后自动填充分类。 |
| **验收标准** | 选择/输入标签后，分类字段自动填充为对应的分类；金额和时间不受影响，仍由用户手动填写。 |

### 新功能 5：消费时间（精确到小时和分钟）

| 项目 | 说明 |
|------|------|
| **背景** | 当前只记录日期（YYYY-MM-DD），用户需要精确到小时和分钟。 |
| **需求** | 在数据模型中增加「消费时间」字段（替代原"创建时间"），格式为 `YYYY-MM-DD HH:mm`。 |
| **展示位置** | - **账单列表页**：仅显示日期 `YYYY-MM-DD`，**不显示**小时和分钟<br>- **账单详情页**：显示完整时间 `YYYY-MM-DD HH:mm` |
| **录入方式** | - 新增记账时，消费时间默认取当前时间（精确到分钟）<br>- 用户可自行修改日期和时间<br>- 支持日期选择器和时间选择器组件 |
| **验收标准** | 新增记录时默认带当前时间可修改；列表页显示日期，详情页显示完整时间。 |

## 4. 界面设计变更

### 4.1 主题色变更：从紫色 → 灰褐色

| 项目 | 说明 |
|------|------|
| **变更内容** | 主色调从紫色系调整为**灰褐色系**，营造冷静、淡雅的视觉风格。 |
| **默认主题** | 默认使用灰褐色主题。 |
| **自定义** | 支持用户自定义主题色（可在设置页面选择主色），但灰褐色为默认出厂设置。 |
| **设计参考** | Material You 动态配色风格，柔和的中性色调，适合长时间使用。 |
| **涉及范围** | 全局颜色变量（primary、secondary、surface、background 等）全部重新定义。 |

### 4.2 导航布局

| 场景 | 导航方式 | 说明 |
|------|---------|------|
| **手机端（安卓 WebView）** | 左侧边栏 **保持不变** | 保持现有实现 |
| **PC 浏览器** | 左侧边栏 | 修复 Bug 1，确保 PC 端也能显示左侧边栏导航 |

左侧边栏布局（从上到下）：
1. **顶部** — 应用图标 + 名称（"Money App 个人记账"）
2. **中间导航菜单**：
   | 图标 | 菜单 | 对应页面 |
   |------|------|---------|
   | 🏠 | 主页 | DashboardPage |
   | 📋 | 账单 | RecordListPage |
   | 📊 | 统计 | StatisticsPage |
   | 💰 | 预算 | BudgetPage |
3. **底部（append 区域）**：
   - **设置**（`mdi-cog-outline`，跳转 SettingsPage）
   - **深色/浅色模式切换开关**（太阳/月亮图标 + switch 控件）

### 4.3 记账入口

记账（新增记录）**不在左侧边栏菜单中**，而是通过**右下角悬浮 FAB 按钮**（`mdi-plus` 加号图标）进入。该按钮在 PC 和手机端均固定在右下角，点击后跳转到记账表单页（RecordFormPage /add）。

### 4.4 账单列表卡片布局（重构后）

```
┌──────────────────────────────────┐
│  [分类图标]  标题（标签/分类名称）    │
│               ¥金额   2026-01-15    │
└──────────────────────────────────┘
```

> 说明：列表页仅显示日期 `YYYY-MM-DD`，详情页才显示完整时间 `YYYY-MM-DD HH:mm`。

### 4.5 深色模式颜色修正

修复深色模式下以下关键元素的对比度问题：

| 元素 | 修复要求 |
|------|---------|
| 主文字颜色 vs 背景色 | 对比度 ≥ 4.5:1 |
| 次要文字颜色 vs 背景色 | 对比度 ≥ 3:1 |
| 卡片背景 vs 页面背景 | 有明显层次区分 |
| 输入框、按钮文字 | 清晰可读 |

## 5. 数据模型变更

### 5.1 Tag 模型变更

| 字段 | 变更类型 | 说明 |
|------|---------|------|
| `category_id` | **新增** | 标签关联的分类（可选），用于快速记账时自动匹配分类 |

变更后的 Tag 模型：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | Integer (PK) | ✅ | 主键，自增 |
| `name` | String | ✅ | 标签名称，如"奶茶"、"共享单车" |
| `category_id` | Integer (FK) | ❌ | 关联分类表，可为 null（未关联分类） |

### 5.2 Record 模型变更

| 字段 | 变更类型 | 说明 |
|------|---------|------|
| `create_time` | **移除** | 不再使用创建时间字段 |
| `consume_time` | **新增** | 消费时间，格式 `YYYY-MM-DD HH:mm`，默认当前时间，用户可修改 |
| `tag_id` | **变更** | 一条记录限制一个标签（原设计为多标签），改为单值外键，可为 null |

### 5.3 变更后的 Record 模型

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | Integer (PK) | ✅ | 主键，自增 |
| `amount` | Decimal(10,2) | ✅ | 金额，正数=收入，负数=支出 |
| `type` | Enum | ✅ | `income`（收入）/ `expense`（支出） |
| `category_id` | Integer (FK) | ✅ | 关联分类表（必填） |
| `tag_id` | Integer (FK) | ❌ | 关联标签表，可为 null（未打标签时为空） |
| `consume_time` | DateTime | ✅ | 消费时间，格式 `YYYY-MM-DD HH:mm` |
| `note` | Text | ❌ | 备注 |

## 6. API 变更说明

### 6.1 新增接口

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/records/{id}` | 获取单条记录详情（供详情页使用） |
| GET | `/api/tags/{id}` | 获取单个标签详情（含关联的分类信息） |

### 6.2 修改接口

| 方法 | 端点 | 变更内容 |
|------|------|---------|
| POST | `/api/records/` | 请求体中 `create_time` 替换为 `consume_time`，`tags`（列表）替换为 `tag_id`（单值，可为 null） |
| PUT | `/api/records/{id}` | 同上 |
| GET | `/api/records/` | 返回数据中增加 `consume_time`（列表页仅日期），移除 `create_time`，`tags` 改为 `tag`（单值对象，含 id、name、关联的 category_id） |
| POST | `/api/tags/` | 请求体中增加可选字段 `category_id`，用于标签关联分类 |
| PUT | `/api/tags/{id}` | 同上，支持修改标签的关联分类 |
| GET | `/api/tags/` | 返回数据中增加每个标签关联的 `category_id` |
| GET | `/api/statistics/` | 修复统计计算逻辑 |

## 附录：变更总结（v1.0 → v1.1）

| 维度 | v1.0 | v1.1 |
|------|------|------|
| 主题色 | 紫色 | 灰褐色（默认，可自定义） |
| 导航 | 手机有侧边栏，PC 无导航 | 手机保持侧边栏，PC 增加侧边栏 |
| 账单列表标题 | 分类名称 | 优先显示标签名称，无标签时回退显示分类名称 |
| 账单列表时间 | 显示完整时间 | 仅显示日期（YYYY-MM-DD） |
| 详情页时间 | 无详情页 | 显示完整消费时间（YYYY-MM-DD HH:mm） |
| 时间字段 | 创建时间（日期） | 消费时间（精确到分钟），默认当前时间可修改 |
| 标签 | 多条记录可对应多个标签 | 一条记录限制一个标签，**非必填** |
| 分类 | 必填 | 必填（保持不变） |
| 标签-分类关联 | 无关联 | 标签可关联一个分类，快速记账时自动匹配 |
| 账单详情 | 无（点击勾选） | 有独立详情页，支持查看和编辑 |
| 深色模式（PC） | 默认深色，无法切换 | 补充切换按钮，与手机端功能统一 |
| 深色模式（手机） | 正常，有切换按钮 | 保持不变 |
| 统计功能 | 金额始终显示为 0 | 修复统计计算逻辑 |
| 快速记账 | 无自动匹配 | 输入标签时自动匹配分类，金额和时间手动填写 |
